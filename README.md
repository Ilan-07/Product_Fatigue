# Product Fatigue Detection System

A multi-modal machine learning system that detects product fatigue across three dimensions — emotional (reviews), commercial (sales), and behavioral (usage) — and fuses them into a unified prediction via a late-fusion meta-model.

## Quick Start

### 1. Environment Setup

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 2. Data Setup

Download the raw datasets and place them in `data/raw/`:

| File | Source |
|------|--------|
| `amazon_reviews.csv` | [Amazon Fine Food Reviews (Kaggle)](https://www.kaggle.com/datasets/snap/amazon-fine-food-reviews) |
| `online_retail.xlsx` | [UCI Online Retail II](https://archive.ics.uci.edu/ml/datasets/Online+Retail+II) |
| `2019-Oct.csv`, `2019-Nov.csv` | [eCommerce Behavior Data (Kaggle)](https://www.kaggle.com/datasets/mkechinov/ecommerce-behavior-data-from-multi-category-store) |

### 3. Run EDA Notebooks (generates processed data)

```bash
jupyter notebook notebooks/01_eda_reviews.ipynb
jupyter notebook notebooks/02_eda_sales.ipynb
jupyter notebook notebooks/03_eda_usage.ipynb
```

These produce the feature CSVs in `data/processed/` that the training pipeline reads.

### 4. Run Training Pipeline

```bash
python3 src/main.py
```

This single command runs the full pipeline end-to-end:

1. Loads processed datasets from `data/processed/`
2. Applies feature stability fixes (log-diffs, epsilon denominators, dead feature removal, correlation pruning at |r| > 0.98)
3. Computes interaction features per modality
4. For each modality (reviews, sales, usage):
   - Collapses labels to binary (healthy / fatigued)
   - Performs walk-forward temporal split (last 3 periods held out)
   - Drops label-derived columns to prevent leakage
   - Trains XGBoost, Random Forest, Logistic Regression with SMOTE inside CV folds
   - Calibrates probabilities (Platt scaling + isotonic regression)
   - Evaluates on temporal test set with SHAP feature importance
5. Runs late fusion (OOF branch probabilities → XGBoost meta-model)
6. Executes ablation studies (feature removal impact analysis)
7. Saves all artifacts

### 5. Run Stress Tests

```bash
python3 src/stress_test.py
```

## Where Things Get Saved

| Artifact | Location | Description |
|----------|----------|-------------|
| **Trained models** | `models/` | Per-modality `.pkl` files (XGBoost, RF, LR, calibrated variants, KMeans) |

| **Fusion metrics** | `outputs/fusion_metrics.json` | Fusion CV/test F1, accuracy |
| **SHAP charts** | `outputs/{modality}_shap_{model}.png` | Feature importance visualizations |
| **Confusion matrices** | `outputs/{modality}_confusion_matrix_{model}.png` | Per-model confusion matrices |
| **ROC curves** | `outputs/{modality}_roc_{model}.png` | ROC curves per model |
| **Ablation reports** | `outputs/ablations/{modality}_ablation_report.json` | Feature ablation impact analysis |
| **Subgroup reports** | `outputs/ablations/{modality}_subgroup_report.json` | Performance by lifecycle stage / age |

## Pipeline Configuration

Key flags in `src/main.py`:

```python
USE_FORWARD_LABELS = False      # Same-window binary labels
BINARY_CLASSIFICATION = True    # healthy vs. fatigued
USE_WALK_FORWARD = True         # Temporal validation
TEST_PERIODS = 3                # Last 3 periods held out
ENABLE_FUSION = True            # Late fusion meta-model
ENABLE_FEATURE_STABILITY = True # Feature safety (log-diffs, correlation pruning)
ENABLE_TEXT_EMBEDDINGS = False  # SBERT disabled (ablation showed -4% F1)
ENABLE_ABLATIONS = True         # Ablation studies
```

## Anti-Leakage Safeguards

| Mechanism | Implementation |
|-----------|---------------|
| Label-derived column dropping | `LABEL_DERIVED_COLS` in `data_loader.py` removes fatigue scores, Z-scores |
| Walk-forward temporal split | Train on past, test on future — no random shuffling across time |
| SMOTE inside CV folds | `imblearn.Pipeline` ensures synthetic samples never leak into validation |
| Scaler isolation | StandardScaler fit on training split only |
| Correlation pruning | Features with \|r\| > 0.98 deduplicated |
| Shift-before-rolling | `shift(1)` applied before rolling windows to prevent lookahead |
